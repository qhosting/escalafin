⚠️  El push requiere autenticación de GitHub

═══════════════════════════════════════════════════════════════

OPCIÓN 1 - TOKEN DE ACCESO (MÁS RÁPIDO)
═══════════════════════════════════════════════════════════════

1. Crear token en GitHub:
   → https://github.com/settings/tokens
   → "Generate new token (classic)"
   → Scope: "repo" (full control)
   → Copiar el token

2. Hacer push con token:
   cd /home/ubuntu/escalafin_mvp
   git push https://TU_TOKEN@github.com/qhosting/escalafin-mvp.git main

   (Reemplazar TU_TOKEN por tu token real)

O usar el script:
   ./comandos-push-rapido.sh TU_TOKEN

═══════════════════════════════════════════════════════════════

OPCIÓN 2 - SSH KEY (MÁS SEGURO)
═══════════════════════════════════════════════════════════════

1. Generar SSH key (si no tienes):
   ssh-keygen -t rsa -b 4096 -C "tu-email@ejemplo.com"

2. Copiar la key:
   cat ~/.ssh/id_rsa.pub

3. Agregar en GitHub:
   → https://github.com/settings/ssh/new
   → Pegar la key

4. Cambiar repo a SSH:
   git remote set-url origin git@github.com:qhosting/escalafin-mvp.git

5. Push:
   git push origin main

═══════════════════════════════════════════════════════════════

OPCIÓN 3 - CREDENTIAL HELPER
═══════════════════════════════════════════════════════════════

1. Configurar:
   git config --global credential.helper store

2. Push (pedirá usuario y token una vez):
   git push origin main
   
   Username: tu-usuario-github
   Password: tu-token (NO tu contraseña)

3. Futuras veces no pedirá credenciales
